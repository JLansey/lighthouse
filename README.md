# Nantucket Lightship / LV-112 — United States Lightship Museum

A modern, fully responsive, static rebuild of the United States Lightship Museum
website for **Nantucket Lightship / LV-112**, a National Historic Landmark berthed
in East Boston.

The content is faithful to the original site — same pages, same words, same photos —
but the markup and styling have been rebuilt from scratch to be **mobile-first,
fast, accessible, and easy to deploy** on Cloudflare Pages.

## What changed

- **Mobile-optimized**: responsive layout, an off-canvas hamburger menu, fluid
  typography, and touch-friendly tap targets. Works from small phones to large desktops.
- **Modern HTML/CSS/JS**: semantic HTML5, a single hand-written stylesheet, and a tiny
  vanilla-JS file (no frameworks, no build dependencies, no tracking).
- **Accessible**: skip link, keyboard-navigable menus and gallery lightbox, `aria` labels,
  reduced-motion support, and proper alt text.
- **Performance**: lazy-loaded gallery images, cache headers, and no external blocking assets.
- **SEO / link preservation**: per-page titles and descriptions, plus `_redirects` that
  301 every legacy `*.htm` URL to its new clean `.html` page so existing bookmarks and
  inbound links keep working.

## Deploy to Cloudflare Pages

This is a plain static site — there is **no build step required**.

### Option A — Dashboard (Git)
1. Push this folder to a GitHub/GitLab repo.
2. In the Cloudflare dashboard: **Workers & Pages → Create → Pages → Connect to Git**.
3. Select the repo. For build settings:
   - **Framework preset:** None
   - **Build command:** *(leave empty)*
   - **Build output directory:** `/`
4. Deploy. Cloudflare automatically applies `_redirects` and `_headers`.

### Option B — Wrangler CLI (direct upload)
```bash
npm install -g wrangler
wrangler pages deploy . --project-name=nantucket-lv112
```

### Custom domain
After the first deploy, add `nantucketlightshiplv-112.org` (and `www`) under the
Pages project's **Custom domains** tab and follow the DNS instructions.

## Local preview

```bash
python3 -m http.server 8000
# then open http://localhost:8000
```

## Editing content

Each page is a self-contained static HTML file you can edit directly. The shared
header/footer chrome for the interior pages is generated by `build_site.py` to keep
it consistent; if you change navigation or the footer, edit that script and run:

```bash
python3 build_site.py
```

(`index.html` and `404.html` are standalone and edited directly.)
